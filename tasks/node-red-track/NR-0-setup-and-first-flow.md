# NR-0 — ติดตั้ง Node-RED + flow แรก 🔧

**ระดับ:** อุ่นเครื่อง | **น่าจะใช้เวลา:** ครึ่งวัน–1 วัน

---

## เป้าหมาย

ลง Node-RED ให้รันได้ + ทำ flow แรกง่ายๆ ให้เห็นว่า "ข้อมูลมันไหลจากกล่องนึงไปอีกกล่องได้จริง" เท่านี้ก็ผ่านแล้วค่ะ

---

## ขั้นที่ 1 — ติดตั้ง

Node-RED รันบน Node.js (ตัวเดียวกับที่น้องใช้ในงาน frontend) ลงครั้งเดียวจบ:

```bash
npm install -g node-red    # ลงแบบ global
node-red                   # สั่งให้มันทำงาน
```

พอรันแล้วมันจะบอกว่า `Server now running at http://127.0.0.1:1880/`
เปิดเบราว์เซอร์ไปที่ **http://localhost:1880** → จะเจอหน้า editor ของ Node-RED (พื้นที่ว่างๆ ตรงกลาง + รายการ node ด้านซ้าย)

> ถ้าลงไม่ผ่าน / ติด permission → capture error ส่งพี่ได้เลย หรือลองดูวิธีลงในเว็บทางการ nodered.org/docs/getting-started/local

## ขั้นที่ 2 — รู้จักหน้าจอคร่าวๆ

- **ซ้าย** = Palette รายการ node (กล่อง) ทั้งหมด ลากมาวางได้
- **กลาง** = พื้นที่ทำ flow (canvas)
- **ขวา** = ข้อมูล/debug
- **ปุ่ม Deploy มุมขวาบน (สีแดง)** = กดทุกครั้งหลังแก้ flow ไม่งั้นมันไม่ทำงาน (จำให้ขึ้นใจเลย 😄)

## ขั้นที่ 3 — flow แรกในชีวิต: inject → debug

นี่คือ "hello world" ของ Node-RED:

1. ลาก node **`inject`** (ชื่ออาจขึ้นว่า "timestamp") มาวางบน canvas — มันคือปุ่มกดเพื่อ "ยิงข้อมูลออกไป"
2. ลาก node **`debug`** มาวาง — มันคือกล่องที่ "โชว์ข้อมูลที่ได้รับ"
3. **ลากเส้นจากจุดด้านขวาของ inject ไปจุดด้านซ้ายของ debug** (นี่คือ "สาย" ที่ข้อมูลจะไหลผ่าน)
4. กด **Deploy**
5. กดปุ่มเล็กๆ ข้าง inject แล้วดูแท็บ **Debug** ด้านขวา → จะเห็นตัวเลข timestamp โผล่มา 🎉

ยินดีด้วย! น้องเพิ่งทำให้ข้อมูลไหลจากกล่องนึงไปอีกกล่องสำเร็จ — Node-RED ทั้งหมดก็คือหลักการนี้ซ้ำๆ แค่ node มันเก่งขึ้นเรื่อยๆ

## ขั้นที่ 4 — ลองเล่นต่ออีกนิด

- ดับเบิลคลิกที่ node `inject` → เปลี่ยนจาก timestamp เป็น **string** แล้วพิมพ์ข้อความ เช่น `"สวัสดีจากไลน์ผลิต"` → Deploy → กดดู debug
- ลองเพิ่ม node **`function`** คั่นกลางระหว่าง inject กับ debug แล้วเขียน JS สั้นๆ:
  ```js
  msg.payload = "เครื่องส่งมาว่า: " + msg.payload;
  return msg;
  ```
  👉 สังเกตคำว่า `msg.payload` — นี่คือ "ก้อนข้อมูล" ที่ไหลในสาย เดี๋ยว Phase ต่อไปเราจะเล่นกับมันเยอะ

## ขั้นที่ 5 — เก็บงานเข้า git

Node-RED เก็บ flow ไว้ให้อัตโนมัติ แต่เราอยากเก็บเข้า repo ด้วย:
- เมนู **☰ (มุมขวาบน) → Export → Download** จะได้ไฟล์ JSON
- เอาไฟล์นั้นใส่ใน `node-red-flows/nr-0-first-flow.json` ใน repo แล้ว commit
