# NR-4 — Capstone: Shop-Floor Live Monitor 🎯🏭

**ระดับ:** กลาง (งานรวมยอด) | **น่าจะใช้เวลา:** 1–2 สัปดาห์

---

## ทำไมต้องทำงานนี้

ถึงตรงนี้น้องผ่านครบแล้ว — ทำ dashboard ได้ (NR-1), อ่าน MES ได้ (NR-2), ส่ง event เข้า MES ได้ (NR-3)
Capstone คือเอาทุกอย่างมารวมเป็น **ระบบเดียวที่ใช้งานได้จริงหน้าไลน์ผลิต**

เป้าหมาย: หน้าจอที่ทีมผลิตเปิดทิ้งไว้แล้วเห็นภาพรวมทั้งไลน์ + ระบบส่งสัญญาณเข้า MES อัตโนมัติ + เตือนเมื่อมีปัญหา

## โจทย์ (ออกแบบรายละเอียดเองได้ ไม่มีเฉลยตายตัว)

สร้าง flow + dashboard ที่:

1. **จำลองหลายเครื่อง** (อย่างน้อย 3 เช่น R1, R2, R3) แต่ละเครื่องมี อุณหภูมิ + สถานะ (RUNNING/IDLE/DOWN)
2. **Dashboard ภาพรวม** — เห็นทุกเครื่องในจอเดียว (gauge/text/ตาราง) จัด layout ให้ดูง่าย
3. **ส่ง event เข้า MES** — ค่าจากแต่ละเครื่อง POST เข้า `/api/machine/event` เป็นจังหวะ (จาก NR-3)
4. **Alert** — ถ้าเครื่องไหนอุณหภูมิเกินเกณฑ์ (เช่น > 85°C) หรือสถานะ DOWN → โชว์เตือนเด่นๆ บน dashboard (สีแดง/ข้อความ)
5. **(ถ้าไหว) อ่านกลับจาก MES** — ดึง `/api/mes/stations/monitor` มาเทียบ/โชว์คู่กัน (จาก NR-2)

## น้องจะได้ฝึกอะไร

- รวมหลาย flow ให้ทำงานด้วยกันอย่างเป็นระเบียบ (จัด flow ไม่ให้รก — ใช้ Link node / แยก Tab ได้)
- ออกแบบ dashboard ที่ "คนหน้างานใช้จริง" — อันนี้สกิล frontend/UX ของน้องได้ใช้เต็มที่
- คิดเรื่อง alert / threshold (logic จริงของระบบ monitoring)

## แนวทางทำ (แบ่งเป็นก้าว อย่าทำทีเดียวหมด)

1. เริ่มจาก 1 เครื่องให้ครบวงจรก่อน (sim → dashboard → POST MES → alert) — ก๊อปจาก NR-1/3
2. พอ 1 เครื่องนิ่งแล้ว ค่อยขยายเป็น 3 เครื่อง
3. จัด layout dashboard ให้สวย (Group/Tab)
4. ใส่ alert logic เป็นอย่างสุดท้าย
5. commit/export ทุกครั้งที่ก้าวนึงเสร็จ

## อยากให้ลองคิดเอง (นี่คือโจทย์ระดับ engineer)

- ถ้าเครื่อง DOWN ควรหยุดส่ง TEMP event ไหม หรือส่งต่อแต่ mark สถานะ?
- threshold 85°C ควร hardcode หรือทำให้ปรับได้จาก dashboard? (dashboard มี node ทำ slider/input ได้)
- ถ้าอยากให้ alert เด้งไปมือถือด้วยล่ะ? (ไม่ต้องทำ แต่ลองคิดว่า Node-RED ทำได้ยังไง — มี node แจ้งเตือนหลายแบบ)

> 💬 Phase นี้ไม่มีคำตอบเดียว พี่อยากเห็น**วิธีคิด**ของน้องมากกว่าความสมบูรณ์ ออกแบบมาแล้วเรามาคุยกันใน PR ว่าจะปรับตรงไหน

## เชื่อมของจริง (ทำกับพี่)

พอ flow นิ่งบน mock/sim แล้ว เราจะนัดเวลากันลองต่อกับ **MES backend จริง** + คุยว่าจะเอาไปใช้หน้าไลน์ได้จริงแค่ไหน อันนี้พี่ช่วยเรื่อง server/network ให้

## เช็คตัวเองว่าใช่รึยัง

- [ ] dashboard โชว์ ≥3 เครื่อง ภาพรวมในจอเดียว ดูเข้าใจง่าย
- [ ] แต่ละเครื่องส่ง event เข้า MES เป็นจังหวะ (จับ error ได้)
- [ ] มี alert เมื่ออุณหภูมิเกิน/เครื่อง DOWN
- [ ] flow จัดเป็นระเบียบ อ่านรู้เรื่อง (ไม่ใช่สายพันกันมั่ว)
- [ ] export flow → `node-red-flows/nr-4-capstone.json` + PR `feat/nr-4-capstone` พร้อม screenshot dashboard + flow + เขียนเล่าวิธีคิดสั้นๆ
