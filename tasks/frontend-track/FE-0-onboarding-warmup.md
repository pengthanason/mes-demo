# FE-0 — ตั้งเครื่องให้พร้อม + อุ่นเครื่อง 🔧

**ระดับ:** อุ่นเครื่อง | **น่าจะใช้เวลา:** 1–2 วัน (ไม่ต้องรีบ)

---

## เป้าหมายของงานนี้

ก่อนจะทำงานอะไรได้ เราต้องทำให้ project **รันขึ้นมาในเบราว์เซอร์ได้ก่อน** แล้วลองแก้อะไรเล็กๆ 1 จุด ให้เห็นกับตาว่า "อ๋อ แก้ตรงนี้แล้วหน้าจอเปลี่ยนจริงด้วย" — เท่านี้ก็ผ่านงานแรกแล้วค่ะ

ฟังดูเหมือนง่าย แต่จริงๆ การตั้ง dev environment ให้รันได้คือด่านที่ developer ทุกคนต้องผ่าน และมันมักจะงงที่สุดในวันแรก **ถ้าน้องติดตรงนี้ ปกติมากๆ** ทุกคนเคยเป็น

> 🎯 จุดที่อยากให้จำ: งานนี้เราสนใจแค่ **frontend** รันได้พอ — **ไม่ต้องสนใจ backend ที่อาจรันไม่ขึ้น** เดี๋ยวพี่อธิบายว่าทำไมด้านล่าง

## ขั้นที่ 1 — ทำให้ frontend รันขึ้นมา

ลองรันตามนี้ใน repo `syntech_mes_draft`:

```bash
cd frontend
npm install      # ติดตั้ง dependency — รอบแรกจะนานหน่อย ปกติค่ะ
npm run dev      # สั่งให้ dev server ทำงาน
```

ถ้าสำเร็จ Vite จะบอก URL มา (ปกติ `http://localhost:5173`) เปิดในเบราว์เซอร์ → ควรเห็นหน้า **"SYNTECH MES"** มีแถบเมนูด้านบน 🎉

### ถ้ามัน error (exit code 1) — ไม่ต้องตกใจ มาไล่ดูกัน

ลองเทียบอาการกับตารางนี้ทีละข้อนะ:

| อาการที่เห็น | น่าจะเพราะ | ลองทำ |
|-------------|-----------|-------|
| `Cannot find module` / หา vite ไม่เจอ | `npm install` ยังไม่ครบ | ลบโฟลเดอร์ `node_modules` กับไฟล์ `package-lock.json` ทิ้ง แล้ว `npm install` ใหม่ |
| ขึ้น error สีแดงเกี่ยวกับ TypeScript | มี type ผิดในไฟล์ `.tsx` บางไฟล์ | อ่าน error ว่าชี้ไฟล์ไหน บรรทัดไหน — บ่อยครั้ง dev server ยังรันต่อได้ ลองเปิดเบราว์เซอร์ดูก่อน |
| `EADDRINUSE` / port ถูกใช้แล้ว | มี process เก่าค้างอยู่ | สั่ง `npm run dev -- --port 5174` ใช้ port อื่นแทน |
| **หน้าเว็บขึ้นแล้ว แต่ console (F12) มี network error สีแดง / data ว่าง** | **backend (:5100) ไม่ได้รัน** | **อันนี้ปกติ ข้ามไปได้เลย** 👇 อ่านกล่องข้างล่าง |

> 💡 **ทำไม backend error ถึงข้ามได้?**
> เว็บเราแยกเป็น 2 ส่วน: **frontend** (หน้าตา ที่น้องดูแล) กับ **backend** (เซิร์ฟเวอร์ที่ส่งข้อมูล)
> งานนี้เราแค่ต้องการให้ **หน้าตา** โหลดขึ้นมาเห็น layout ก็พอ — ส่วนข้อมูลจริงเป็นเรื่องของ backend ซึ่งไม่ใช่งานน้องในตอนนี้
> ถ้าเห็นแถบเมนู SYNTECH MES ขึ้นมา = **น้องผ่านขั้นที่ 1 แล้ว** ถึงข้อมูลจะว่างก็ไม่เป็นไรเลย

**ลองตามตารางแล้วยังไม่ขึ้นภายใน ~2 ชั่วโมง?** capture หน้าจอ terminal ที่ขึ้น error แล้วส่งมาให้พี่ดูเลยค่ะ อย่านั่งงมคนเดียวนาน นี่คือสิ่งที่พี่อยากช่วย 🙂

## ขั้นที่ 2 — แก้ของเล็กๆ 1 จุด ให้เห็นว่ามัน "มีชีวิต"

ตอนนี้ project รันแล้ว มาลองแตะมันดูบ้าง เลือกทำ **1 อย่าง** ที่น้องอยากลอง:

- **(a)** ทำให้เมนูที่ active (หน้าที่กำลังเปิดอยู่) เด่นขึ้น — เปิด `src/App.jsx` หาส่วน `NavLink` ลองเปลี่ยนสีหรือเพิ่มเส้นใต้
- **(b)** เพิ่ม footer เล็กๆ ใต้เนื้อหา เขียนว่า `© 2026 Synergy Technology` จัดกึ่งกลาง สีจางๆ (ลองใช้ `var(--text-muted)`)
- **(c)** เปลี่ยน title ของเว็บใน `frontend/index.html` (ดู `<title>`)

ไม่มีถูกผิดนะ จุดประสงค์คือให้น้อง**กล้าแก้** แล้ว save → กลับไปดูเบราว์เซอร์ → เห็นมันเปลี่ยน นั่นแหละคือ feedback loop ของ frontend dev ที่น้องจะใช้ไปตลอด

> ลองสังเกตว่าพอ save ไฟล์ หน้าเว็บมัน refresh เองเลยไม่ต้องกด F5 — อันนั้นเรียกว่า Hot Reload เป็นของขวัญจาก Vite 🎁
