# FE-4 — Daily Production Report Table (แบบที่พี่ Kotchapat ทำใน Outlook) 📋

**ระดับ:** เริ่ม–กลาง (ต่อยอดจาก FE-1) | **น่าจะใช้เวลา:** 4–6 วัน

---

## ทำไมต้องทำงานนี้ (งานนี้ทีมใช้จริงทุกวัน)

ทุกวันพี่ **Kotchapat** (Process & Production Engineer) ส่ง **"Production Report"** เข้า Outlook ให้ทีม — เป็นสรุปสถานะงานผลิตแต่ละโปรเจกต์ เช่น:

```
Project Status :
1. E13A_STD : THS
   - ทดสอบการทำงาน จำนวน 270 pcs
   - กำหนดการส่งมอบ 30 Mar 2026
2. ZSZ003-081A : TAD
   - ผลิต SMT เสร็จ 1,200 pcs เหลือตรวจสอบ/Depanel/Packing
3. 5K45 : THS
   - Depanel PCBA, ส่งมอบ 27 Mar 2026
```

ตอนนี้มันอยู่ในอีเมล (พิมพ์มือทุกวัน อ่านย้อนยาก ทำสถิติไม่ได้) — เราอยากให้มันเป็น **ตารางในระบบ MES** ที่ทุกคนเปิดดูสถานะล่าสุดได้ กรองได้ และต่อไปดึงข้อมูลจริงจาก MES มาเติมได้

งานน้องคือทำ **หน้าตาราง Daily Production Report** นั้น — ต่อยอดตรงจากตาราง history ที่น้องทำใน FE-1 เลย

> 🎯 เริ่มด้วยข้อมูลตัวอย่าง (mock) ก่อน เหมือนทุกงาน แล้วค่อยต่อข้อมูลจริงทีหลัง
> 📌 **ก่อนเริ่ม: ขอ template ตารางจริงจากพี่/พี่ Kotchapat** ว่าเขาอยากได้คอลัมน์อะไรบ้างเป๊ะๆ (ข้างล่างพี่ใส่ชุดเริ่มต้นให้ แต่ของจริงยืนยันกับพี่ก่อนนะ)

## น้องจะได้ฝึกอะไร

- ต่อยอด component ตาราง (จาก FE-1) ให้ใหญ่ขึ้น มีหลายคอลัมน์
- ทำ **filter / search** (กรองตามลูกค้า, สถานะ, วันส่งมอบ)
- ทำ **status badge** + แสดงวันที่ให้อ่านง่าย
- (โบนัส) สรุปยอดด้านบน เช่น จำนวนโปรเจกต์ที่ส่งมอบสัปดาห์นี้

---

## คอลัมน์เริ่มต้น (ยืนยันกับพี่ก่อนได้)

| คอลัมน์ | ตัวอย่าง | หมายเหตุ |
|---------|----------|----------|
| No. | 1, 2, 3 | ลำดับ |
| Project Code | E13A_STD | รหัสงาน |
| Customer | THS / TAD | ลูกค้า |
| Status | "ทดสอบการทำงาน" | สถานะความคืบหน้า |
| Qty (pcs) | 270 | จำนวน |
| Delivery Date | 30 Mar 2026 | กำหนดส่งมอบ |
| Stage (โบนัส) | SMT / Test / Packing | ขั้นตอนปัจจุบัน |

## มาเริ่มกันทีละขั้น

**ก้าวที่ 1 — หน้า + route**
สร้าง `frontend/src/pages/ProductionReportPage.tsx` + route `#/production-report` + เมนู

**ก้าวที่ 2 — ตารางจาก mock (เหมือน FE-1)**
เอา `SAMPLE_REPORT` มา `.map()` เป็นแถวตาราง แยกแถวเป็น component `ReportRow.tsx` ถ้าอยาก reuse

**ก้าวที่ 3 — status badge + จัดวันที่**
- `stage` ทำเป็น badge สี (SMT / Test / Packing / Depanel คนละสี)
- `delivery` แสดงให้อ่านง่าย + ถ้าเลยกำหนดแล้วให้เป็นสีแดงเตือน (ลองคิดวิธีเช็ควันที่)

**ก้าวที่ 4 — filter / search**
- ช่องค้นหาตาม Project Code
- dropdown กรองตาม Customer (THS / TAD / ...)
- (โบนัส) กรองเฉพาะที่ส่งมอบภายในสัปดาห์นี้

**ก้าวที่ 5 — สรุปด้านบน**
แถบสรุป เช่น "ทั้งหมด 4 โปรเจกต์ · ส่งมอบสัปดาห์นี้ 2 · รวม 1,560 pcs"

**ก้าวที่ 6 (ต่อยอดของจริง — ทำกับพี่)**
ข้อมูลจริงควรมาจากไหน? มี 2 ทาง คุยกับพี่เลือกได้:
- ให้ทีมกรอกรายงานในหน้านี้เอง (มีฟอร์มเพิ่มแถว) แทนพิมพ์ในอีเมล
- หรือดึงจากข้อมูล WO ใน MES มาเติมอัตโนมัติ (ส่วนนี้พี่ช่วยเรื่อง API)

## อยากให้ลองคิดเอง

- ตารางนี้ถ้ามี 50 โปรเจกต์จะอ่านยากไหม? ควรมี filter/จัดกลุ่มยังไงให้ดูรู้เรื่อง
- "เลยกำหนดส่งมอบ" ควรเด่นแค่ไหน? (ผู้บริหารอยากเห็นของที่ใกล้ครบกำหนด/เลยกำหนดก่อน)
- ถ้าจะให้ export เป็นไฟล์/ปริ้นได้เหมือนแปะในอีเมล จะทำยังไง (ยังไม่ต้องทำ แค่คิด)
